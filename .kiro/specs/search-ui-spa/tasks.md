# Implementation Plan

- [x] 1. バックエンドStaticFiles統合
- [x] 1.1 FastAPI静的ファイルマウントとDocker設定
  - SearchサービスのFastAPIアプリに静的ファイル配信機能を追加する
  - APIルート定義の後にルートパスへのマウントを設定し、既存エンドポイントとの優先順位を確保する
  - `html=True`オプションでindex.htmlの自動配信を有効にする
  - Dockerfileに静的ファイルディレクトリのコピーを追加する
  - 動作確認用の最小限のHTMLファイルを配置し、ブラウザからアクセスできることを検証する
  - _Requirements: 7.1, 7.2_

- [ ] 2. SPA基盤構築
- [x] 2.1 (P) HTMLスケルトンとCSSレスポンシブレイアウト
  - SPAのエントリポイントとなるHTMLファイルを作成し、検索フォーム・結果表示・クエリ情報の各セクションのDOM構造を定義する
  - CSS Gridを使用したレスポンシブグリッドレイアウトを実装し、デスクトップでは`auto-fill`/`minmax`で列数を自動調整する
  - 768px未満のモバイルビューでは1〜2列に切り替わるメディアクエリを設定する
  - CSS Custom Propertiesでカラーテーマとスペーシングの基本変数を定義する
  - カード内のサムネイル画像が`object-fit: cover`でアスペクト比を維持する設定を含める
  - ローディングスピナー、エラーメッセージ、空結果メッセージのスタイルを定義する
  - フィルタタグ用のバッジスタイル（motif_tags用緑系、color_tags用青系）を定義する
  - _Requirements: 1.1, 2.1, 5.2, 6.1, 6.2, 6.3, 7.1, 7.3_

- [x] 2.2 (P) APIクライアントモジュール
  - Search APIへのPOSTリクエストを送信するfetchラッパーを実装する
  - AbortControllerを使用した30秒タイムアウト制御を組み込む
  - HTTPステータスに基づくエラー分類を実装する（4xx: バリデーション、5xx: サーバーエラー）
  - ネットワーク接続エラーとタイムアウトを個別のエラーメッセージで区別する
  - リクエスト送信時に前回のリクエストをキャンセルする仕組みを実装する
  - _Requirements: 1.2, 4.3, 4.4_

- [x] 3. 検索フォームコンポーネント
- [x] 3.1 検索入力、バリデーション、送信制御
  - テキスト入力フィールドと検索ボタンを持つフォームコンポーネントを実装する
  - inputイベントで空文字チェックを行い、空の場合は検索ボタンを無効化する
  - formのsubmitイベント（Enterキー・クリック共通）で検索コールバックを発火する
  - テキスト入力に500文字の上限を設定し、超過時にユーザーへ通知する
  - 検索処理中はボタンを無効化し、テキストを「検索中...」に変更してフィードバックを提供する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 4.1, 4.2_

- [x] 4. 検索結果表示コンポーネント
- [x] 4.1 (P) 結果グリッドの描画とステータス表示
  - 検索レスポンスを受け取り、結果アイテムをグリッドレイアウトに描画するコンポーネントを実装する
  - 検索結果の件数を「N件の作品が見つかりました」形式でヘッダーに表示する
  - 結果が0件の場合は「該当する作品が見つかりませんでした」メッセージを表示する
  - 検索処理中にローディングスピナーを表示する
  - APIエラー発生時にエラーメッセージを表示し、再試行を促すテキストを含める
  - タイムアウト時は「接続がタイムアウトしました」を表示する
  - _Requirements: 2.1, 2.3, 2.4, 4.1, 4.3, 4.4, 6.1, 6.2_

- [x] 4.2 (P) 結果カードの描画とヒット理由表示
  - 個別の検索結果アイテムを表示するカードコンポーネントを実装する
  - サムネイル画像、作品タイトル、アーティスト名を表示する
  - match_reasonsをカード下部に小さなバッジ/タグとして表示する
  - ホバー（デスクトップ）またはタップ（モバイル）でスコア値とヒット理由の詳細をオーバーレイ表示する
  - サムネイル画像の読み込みエラー時にプレースホルダー画像を表示するフォールバックを設定する
  - _Requirements: 2.2, 3.1, 3.2, 6.3_

- [x] 4.3 (P) クエリ解析結果の表示
  - 検索レスポンスのparsed_queryからsemantic_queryをテキストとして表示するコンポーネントを実装する
  - motif_tagsを緑系バッジ、color_tagsを青系バッジで視覚的に区別して表示する
  - フィルタ条件が空の場合はコンポーネント全体を非表示にする
  - _Requirements: 5.1, 5.2_

- [x] 5. App統合と動作検証
- [x] 5.1 全コンポーネントの接続と状態管理
  - Appエントリポイントでアプリケーション状態（query, loading, response, error）を管理するモジュールを実装する
  - DOMContentLoadedで各コンポーネントを初期化し、イベントチェーンを接続する
  - SearchFormのonSearch → ApiClient.search → ResultGrid描画 + QueryInfo描画のフローを接続する
  - エラー発生時に適切なUIフィードバック（バリデーション、API、タイムアウト、ネットワーク）が表示されることを検証する
  - ページ遷移なしで検索と結果表示が同一画面で完結することを確認する
  - _Requirements: 4.3, 4.4, 7.3_
